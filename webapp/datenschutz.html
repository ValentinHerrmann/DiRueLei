Datenschutz bei DiRueLei

🔒 Lokale Verarbeitung & Optionale Artemis-Integration

DiRueLei ist eine Web-Anwendung, die standardmäßig im Browser ausgeführt wird. Im reinen Offline-Modus werden alle Daten ausschließlich lokal auf Ihrem Gerät verarbeitet.

  • Standardmäßig keine Server-Uploads: Bei lokaler Nutzung werden weder CSV-Dateien noch PDF-Dokumente auf einen Server hochgeladen.
  • Optionale Artemis-Integration: Wenn Sie den JWT-basierten Import/Upload-Flow für Artemis nutzen, werden ausgehende Verbindungen zu Artemis sowie einem Drittanbieter-CORS-Proxy aufgebaut. Hierbei werden erforderliche Daten (wie Tokens, Kurs-IDs und Rückgaben) übertragen.
  • Keine Cloud-Speicherung: Die generellen Verarbeitungsschritte finden weiterhin lokal im Browser statt.
  • Keine Tracking-Cookies: DiRueLei verwendet keine Tracking- oder Analyse-Tools.

💼 Empfehlung: Nutzung auf Dienstgeräten

Wichtig: Lehrkräfte sollten DiRueLei ausschließlich auf schulischen Dienstgeräten verwenden.

  • Private Geräte sollten nicht für die Verarbeitung von Schülerdaten verwendet werden.
  • Stellen Sie sicher, dass das verwendete Dienstgerät den schulischen IT-Sicherheitsrichtlinien entspricht.
  • Nach der Nutzung können alle erzeugten Dateien lokal gelöscht werden.

📋 Welche Daten werden verarbeitet?

DiRueLei verarbeitet folgende Daten primär lokal in Ihrem Browser:

  • CSV-Dateien: Schüler-IDs und Namen aus Mebis-Exporten.
  • PDF-Dateien: Gescannte Dokumente mit QR-Codes.
  • QR-Codes: Automatisch generierte Identifikationscodes.
  • Artemis-Daten (Optional): Bei Nutzung der Artemis-Integration werden JWT-Tokens, Kurs-IDs sowie Rückgabedokumente verarbeitet und an Artemis (ggf. über einen Drittanbieter-CORS-Proxy) gesendet.

Ohne Nutzung der Artemis-Funktionen bleiben alle Daten ausschließlich auf Ihrem Gerät und werden nicht übertragen.

🗑️ Datenlöschung

Da DiRueLei keine Daten speichert oder überträgt, können Sie jederzeit:

  • Den Browser-Tab schließen (alle Daten werden aus dem Arbeitsspeicher entfernt).
  • Die erzeugten PDF-Dateien lokal von Ihrem Gerät löschen.
  • Den Browser-Cache leeren, falls gewünscht.

🔐 Technische Sicherheit

  • Moderne Browser-Sicherheitsmechanismen (Cross-Origin Isolation).
  • Keine Persistierung von Daten über die Sitzung hinaus.
  • Verwendung von WebAssembly (Pyodide) für sichere Python-Ausführung im Browser.
  • Keine Netzwerkverbindungen während der Datenverarbeitung (außer initialem Laden der Anwendung und optionalen Anfragen an Artemis bzw. einen CORS-Proxy).
